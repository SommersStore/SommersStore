# Skill Playbook: SKL-QRO-08 - RedundancyNoveltyScan

## Objetivo Operacional
Separar novidade util de redundancia para evitar inchaco da base do clone.

## Procedimento
1. Quebrar o nectar em claims, regras ou frameworks.
2. Comparar cada item com a base atual.
3. Classificar cada item como `NOVO_PREMIUM`, `NOVO_UTIL`, `REFORCO_UTIL`, `REDUNDANTE` ou `CONFLITANTE`.
4. Recomendar manter, comprimir, arquivar ou remover.

## Saida Esperada
- matriz de claims
- nivel de novidade
- plano de compressao
