# Story 2.4 - Logs com project_id explicito no backend

**Status:** Done  
**Epic:** 2 - MVP do Painel Unico Operacional  
**Criado em:** 2026-04-17  
**Agente responsavel:** @dev (Codex)

---

## Descricao

Como operador do painel, quero que cada log registre `project_id` no backend para a timeline do Master Hub filtrar historico por projeto com confianca, sem depender de inferencia textual.

---

## Criterios de Aceitacao

- [x] `appendExecutionLog` persiste `project_id` no `execution_log.json`.
- [x] Backend aceita `project_id` nos payloads de log e fluxos operacionais.
- [x] Quando `project_id` nao for enviado, backend tenta inferir pelo `target/details` com base em `project_flows`.
- [x] Timeline do Master Hub prioriza filtragem por `entry.project_id`.
- [x] Quality gates cobrem os novos pontos de integracao.

---

## Tarefas
