# Story 1.4 — Automação de Memória e Painel de Continuidade

**Status:** Done  
**Epic:** 1 — Estabilização Operacional do Control Fabric  
**Criado em:** 2026-04-15  
**Agente responsável:** @dev (Codex)

---

## Descrição

Como operador do SommersStore, quero que o painel execute startup/shutdown de memória automaticamente (Oracle/Scribe), para não perder contexto entre sessões e manter continuidade confiável independentemente do modelo usado.

---

## Critérios de Aceitação

- [x] Painel possui rotinas explícitas de startup (`/api/session/start`) e shutdown (`/api/session/close`).
- [x] Startup gera context flash com leitura em cascata de memória.
- [x] Shutdown grava checkpoint, mutação e atualização de estado atual.
- [x] `task.md` e docs de memória humana recebem atualização automática no shutdown.
- [x] Camadas de memória são exibidas no painel (estado, loops, checkpoints, decisões, runs, artefatos).
- [x] Bugs críticos de continuidade no painel foram mitigados (editor path/title, personas/agentes, cores de squads).
