# Story 2.83 - Correcao do filtro de projeto em Master Clones

**Status:** Done
**Epic:** 2 - MVP do Painel Unico Operacional
**Criado em:** 2026-06-15
**Agente responsavel:** @dev + Codex

---

## Descricao

Como operador do AIOX Master, quero alternar entre todos os clones e somente os clones do projeto ativo mesmo depois de abrir um clone no inspector.

---

## Contexto

O filtro por projeto funcionava quando nenhum clone estava selecionado. Ao clicar em um clone, o inspector passava a renderizar apenas o pipeline daquele clone e o botao `Filtrar Projeto` desaparecia, impedindo voltar da visao global para a visao do projeto sem limpar a selecao por outro caminho.

Durante a validacao visual, tambem apareceu um segundo sintoma: ao abrir diretamente `Master > Pajero AI > Clones`, o painel podia renderizar antes de `registry` e `persona assets` terminarem de carregar, exibindo falsamente `Catalogo antes dos filtros: 0`.

---

## Criterios de Aceitacao

- [x] Manter o alternador `Mostrar Todos` / `Filtrar Projeto` visivel no inspector de clone selecionado.
- [x] Persistir o modo todos/projeto no estado local do Master.
- [x] Ao filtrar a partir de um clone selecionado, limpar a selecao e renderizar somente os clones do projeto ativo.
- [x] Preservar a contagem esperada do Pajero: 12 clones filtrados e 30 clones globais.
- [x] Evitar falso estado vazio enquanto o catalogo ainda esta hidratando.
- [x] Rodar quality gates principais.
