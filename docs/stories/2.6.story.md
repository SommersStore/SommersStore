**Status:** Done

## Descricao

Como operador do painel, quero que os eventos de sessao gravem `project_id` automaticamente e que exista um backfill para logs antigos, para manter a timeline do Master Hub util por projeto mesmo apos retomadas.

---

## Criterios de Aceitacao

- [x] Frontend envia `project_id` no ciclo de sessao (`start`, `close`, `pulse` e auto-close).
- [x] Backend normaliza `project_id` no ciclo de sessao e persiste no arquivo de sessao.
- [x] Inferencia de projeto considera aliases historicos (`SIZ` -> `sais`).
- [x] Existe script de manutencao para backfill de `project_id` nos logs.
- [x] Quality gates cobrem o novo fluxo.

---

## Tarefas

- [x] Criar helper de projeto atual no dashboard e propagar no payload de sessao.
- [x] Normalizar e persistir projeto da sessao no backend.
- [x] Reforcar inferencia textual de projeto no backend.
- [x] Adicionar script de backfill com inferencia por texto e sessoes arquivadas.
- [x] Expor comando npm para executar o backfill.
- [x] Atualizar testes de integracao/qualidade do Master Hub.
