# Story 2.110 - Financas Mobile Cloud Inbox 24h

**Status:** Done  
**Epic:** 2 - MVP do Painel Unico Operacional  
**Criado em:** 2026-07-21  
**Executor:** @dev  
**Origem:** correcao solicitada pelo usuario apos perceber que a aba Mobile local fica inacessivel quando o notebook desliga.

---

## Story

**Como** operador da Planilha de Financas,  
**quero** um app mobile hospedado em nuvem para registrar lancamentos 24h,  
**para que** eu possa lancar despesas/receitas/dividas pelo celular mesmo com o notebook desligado e depois importar/conferir no painel local.

---

## Acceptance Criteria

1. Existe uma rota publica hospedavel em Firebase Hosting para `Financas Mobile Cloud`, separada do painel local.
2. O app cloud nao depende do servidor `start_painel.bat` para abrir no celular.
3. O app cloud usa Firebase Web SDK modular com Auth e Firestore quando as variaveis `NEXT_PUBLIC_FINANCAS_FIREBASE_*` estao configuradas.
4. Sem configuracao Firebase, a tela mostra modo setup e nao promete sincronizacao 24h.
5. O formulario cloud registra tipo (`receitas`, `despesas`, `dividas`), nome, valor, data, parcelas e destino da planilha.
6. Os lancamentos sao gravados como pendentes em uma colecao por usuario, sem credenciais hardcoded no codigo.
7. O painel local Financas > Mobile exibe um bloco `Cloud Inbox` explicando a diferenca entre app local e app 24h.
8. O painel local consegue carregar lancamentos pendentes da nuvem quando o usuario informa a config Firebase e faz login.
9. Cada lancamento pendente pode ser aplicado a um destino da planilha local usando a mesma regra do Mobile existente.
10. A aplicacao local marca o lancamento cloud como importado no Firestore quando possivel.
11. O fluxo preserva a conferencia no notebook: nada entra na planilha principal sem acao explicita de importacao.
12. Quality gates do repo raiz continuam passando.

## Dev Notes

- A aba Mobile atual e local: `docs/aiox_dashboard.html` detecta `/mobile` e usa o servidor local para carregar/salvar `projects/financas/data/fin2_data.json`.
- A correcao arquitetural e criar uma captura cloud independente do notebook, usando Firebase Hosting + Cloud Firestore + Firebase Auth.
- Referencia oficial consultada em 2026-07-21: Firebase recomenda SDK Web modular via npm e inicializacao com `initializeApp`; Firestore armazena dados em documentos/colecoes; Hosting publica web apps em CDN global; Firestore pode trabalhar com persistencia offline no cliente.
- A planilha local continua sendo fonte de conferencia; o cloud app atua como inbox pendente.
- Variaveis previstas para build/deploy:
  - `NEXT_PUBLIC_FINANCAS_FIREBASE_API_KEY`
  - `NEXT_PUBLIC_FINANCAS_FIREBASE_AUTH_DOMAIN`
  - `NEXT_PUBLIC_FINANCAS_FIREBASE_PROJECT_ID`
  - `NEXT_PUBLIC_FINANCAS_FIREBASE_STORAGE_BUCKET`
  - `NEXT_PUBLIC_FINANCAS_FIREBASE_MESSAGING_SENDER_ID`
  - `NEXT_PUBLIC_FINANCAS_FIREBASE_APP_ID`
- Colecao proposta: `users/{uid}/financasMobileInbox/{entryId}` com `status: "pending" | "imported"`.
- O painel local tambem usa config digitavel/localStorage para evitar hardcode de chaves ou senha.
