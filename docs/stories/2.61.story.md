# Story 2.61 - Dividas Bancarias do Holerite Alimentam Dividas

**Status:** Done
**Epic:** 2 - MVP do Painel Unico Operacional
**Criado em:** 2026-06-03
**Agente responsavel:** @dev (Codex)

---

## Descricao

Como operador do AIOX, quero que os emprestimos bancarios descontados no holerite PM sejam copiados automaticamente para a secao `Dividas` da `Planilha`, para que novas dividas bancarias surgidas no holerite entrem no controle sem lancamento manual.

---

## Criterios de Aceitacao

- [x] Descontos bancarios extraidos do holerite PM criam ou atualizam linhas em `Planilha > Dividas`.
- [x] Novas dividas bancarias do holerite sao acrescentadas automaticamente quando nao existe linha correspondente.
- [x] Linhas ja existentes sao reaproveitadas por nome canonico e valor mensal aproximado, evitando duplicidade basica.
- [x] `Dividas/Acordos` continua sincronizado com `Planilha > Dividas`.
- [x] Valores extraidos de contrato em `Dividas/Acordos` continuam prevalecendo sobre valores vindos do holerite.
- [x] Regras da Story 2.60 continuam ativas: bancos ficam visiveis/vermelhos em `Despesas PM` e fora da soma de despesas.
- [x] Quality gates e validacao Puppeteer executados.

---

## Tarefas

- [x] Criar sincronizador de dividas bancarias a partir das series de descontos do holerite PM.
- [x] Identificar chave automatica `pmdebt-*` para cada desconto bancario extraido.
- [x] Criar/atualizar linhas em `FIN2_SHEET.dividas`.
- [x] Criar/atualizar credores em `FIN2_CREDITORS` com `source: 'payslipDebt'`.
- [x] Preservar `valuesSource: 'contract'` quando houver contrato extraido.
- [x] Rodar sincronizacao do holerite antes do sincronismo normal `Planilha > Dividas/Acordos`.
- [x] Atualizar testes estaticos.
- [x] Validar no navegador.
- [x] Rodar lint, typecheck e teste.
