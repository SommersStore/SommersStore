# Story 2.93 - Blindagem de Persistencia da Aba Financas

## Status
Done

## Contexto
O usuario relatou que configuracoes recentes da aba `Financas`, especialmente a copia da celula do mes anterior e o salvamento automatico, pareciam ter sido perdidas. Tambem havia mensagem visual `erro ao salvar` no canto superior direito e duvida sobre os botoes `Flash`, `Salvar Tudo` e `Conectar Pasta`.

## Acceptance Criteria
- Confirmar se a regra de heranca mensal da `Planilha` ainda existe e funciona no navegador.
- Investigar se ha copia local mais nova de `fin2_data.json` ou `finance_state.json` para recuperar valores recentes.
- Reforcar o autosave da `Planilha` para salvar localmente antes de depender do servidor.
- Fazer falha do `Conectar Pasta` cair para salvamento via servidor em vez de bloquear a gravacao.
- Garantir que `Salvar Tudo` tente salvar `Financas` antes de acionar GitHub/Firebase.
- Validar os botoes de cabecalho sem executar push/deploy indevido.

## Tasks
- [x] Ler memoria, task board e historico fisico de Financas.
- [x] Criar backup local pre-auditoria dos dados financeiros atuais.
- [x] Testar `/api/save` diretamente no servidor local.
- [x] Validar `Financas > Planilha` via Puppeteer em `http://localhost:4000/`.
- [x] Reforcar `fin2SaveData`, `fin2LoadData` e `fin2TriggerAutoSave`.
- [x] Adicionar fallback do File System Access API para servidor.
- [x] Fazer `Salvar Tudo` abortar se a aba Financas nao salvar antes.
- [x] Atualizar testes de qualidade.
- [x] Rodar lint, typecheck, test e verificacoes de navegador.

## Resultado
- A regra mensal `null = copia o mes anterior` foi confirmada em runtime, com 387 celulas herdadas renderizadas.
- Nao foi encontrada copia externa mais nova do arquivo de dados financeiros; o `fin2_data.json` fisico estava defasado desde `2026-06-14` antes do teste controlado.
- O autosave agora cria backup imediato em `localStorage` e depois grava no servidor com debounce curto.
- Se a permissao/pasta conectada expirar ou falhar, o painel desconecta a pasta e usa o servidor local como fallback.
- `Salvar Tudo` agora tenta salvar Financas primeiro; se falhar, nao segue para GitHub/Firebase.
- `Conectar Pasta` foi documentado no proprio tooltip como pasta local direta, nao como nuvem.
