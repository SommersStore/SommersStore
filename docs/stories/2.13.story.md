# Story 2.13 - Handshake de Continuidade em Novas Conversas

**Status:** Done  
**Epic:** 2 - MVP do Painel Unico Operacional

**Agente responsavel:** @dev (Codex)

## Descricao

Como operador do painel, quero que cada nova conversa tenha sinalizacao explicita do contexto carregado e use o checkpoint mais relevante (nao apenas o ultimo auto-close), para retomar trabalho sem ambiguidade.

---

## Criterios de Aceitacao

- [x] Startup Oracle passa a gerar um artefato unico de retomada (`docs/memory/startup_context_latest.md`).
- [x] `contextFlash` prioriza checkpoint acionavel e registra estrategia de resolucao de contexto.
- [x] Aba Memory mostra `Proxima acao` no bloco de contexto atual.
- [x] Regra operacional do projeto exige handshake de continuidade na primeira resposta de chat novo.
- [x] Quality gates (`lint`, `typecheck`, `test`) executados com sucesso.
