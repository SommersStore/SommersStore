# Story 2.77 - Refinar Dividas e Acordos de Financas

**Status:** Done
**Epic:** 2 - MVP do Painel Unico Operacional
**Criado em:** 2026-06-09
**Agente responsavel:** @dev (Codex)

---

## Descricao

Como operador da aba Financas, quero simplificar `Dividas/Acordos`, aceitar varios documentos por divida, preservar os campos essenciais de parcelas e garantir que o holerite prevaleca sobre contratos nas dividas vindas do holerite.

---

## Criterios de Aceitacao

- [x] O controle de status mostra apenas `Ativo`, `Negociado` e `Suspenso`.
- [x] Campos de prazo inicial e prazo final deixam de aparecer no drawer.
- [x] O upload de contrato/documentos aceita multiplos arquivos selecionados ou arrastados.
- [x] A area antes chamada `Plano de Quitacao` passa a ser `Observacoes`.
- [x] O botao `Marcar Pendurada` deixa de aparecer no drawer.
- [x] A grade de parcelas marca/desmarca apenas a parcela clicada, sem marcar automaticamente as anteriores.
- [x] Contratos continuam podendo extrair parcela, saldo, oferta e quantidade de parcelas.
- [x] Para dividas vindas do holerite, o valor mensal do holerite prevalece sobre o contrato.
- [x] Quality gates e validacao no painel local executados.

---

## Tarefas

- [x] Simplificar os controles do drawer de `Dividas/Acordos`.
- [x] Trocar status para a lista restrita definida pelo usuario.
- [x] Converter anexo unico em lista de documentos por credor.
- [x] Processar multiplos arquivos no input e no drag-and-drop.
- [x] Alterar controle de parcelas para lista individual de parcelas pagas.
- [x] Ajustar regra contrato vs holerite para dividas de origem `payslipDebt`.
- [x] Atualizar testes estaticos.
- [x] Validar no painel local e rodar lint/typecheck/test.
