# Coding And Documentation Standards

- Usar nomes em kebab-case para arquivos de squad.
- Manter dados estruturados em JSON quando forem consumidos pelo painel.
- Manter documentos tecnicos em Markdown com tabelas de status, evidencia e observacoes.
- Toda recomendacao mecanica deve indicar estado: confirmado, provavel ou pendente de validacao.
- Toda especificacao de torque, fluido, capacidade, peca e procedimento deve ter fonte.
- Nao misturar motor, cambio, transferencia, diferencial dianteiro e diferencial traseiro.
