- **Protocolo VEG (Visão, Link, Arquitetura, Estilo, Gatilho):** Framework para criar automações e aplicativos robustos, priorizando confiabilidade sobre velocidade.
    - **Visão (V):** Definir o resultado desejado, serviços internos/integrações e forma de entrega.
    - **Link (L):** Testar todas as conexões de API e credenciais antes de construir o código principal.
    - **Arquitetura (A):** Construir as três camadas do aplicativo (Diretório, Extração, Execução).
    - **Estilo (E):** Criar a interface do usuário (UI) ou formatar saídas da automação.
    - **Gatilho (G):** Definir o que dispara a automação (cron, webhooks, etc.) e preparar para deploy 24/7.

- **`gemini.md` como "Lei":** Arquivos `gemini.md` servem como regras e "habilidades" específicas para um projeto, guiando o comportamento da IA.

## Frameworks
- **Protocolo VEG:** Metodologia estruturada para o desenvolvimento de automações e aplicativos (Visão, Link, Arquitetura, Estilo, Gatilho).
- **Protocolo Zero:** Framework para a inicialização e estruturação de projetos, garantindo a preparação do contexto antes da codificação.
- **Autocorreção/Loop de Reparo:** Sistema de feedback e aprendizado contínuo para a IA baseado na detecção e correção de erros.
- **MCP (Model Context Protocol):** Padrão para habilitar a IA a interagir de forma autônoma e precisa com APIs e Webhooks de outras plataformas.
- **`gemini.md` como "Skills/Habilidades":** Utilização de arquivos Markdown como "documentos de lei" ou "habilidades" para guiar o comportamento e a estética da IA em projetos específicos.
- **Regras Globais do Antigravery:** Conjunto de diretrizes de alto nível que se aplicam a todos os projetos da IA, padronizando o comportamento.
- **Arquitetura de 3 Camadas (Implícita):** Estrutura organizacional das automações (Diretório, Extração, Execução).
